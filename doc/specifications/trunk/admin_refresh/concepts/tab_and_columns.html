    Features:
    1. Moves all the controls for the current page to the top to avoid scrolling.
    2. Hides right menu by default to hide features that are not relevant for a editor (or even most admins) during normal use.
    3. Slims down the top header considerably to get more space for content area, also implements support for small screens (down to 200px) by automatically wrapping the header.

    Also uses CSS3 features to reduce use of graphic, so looks best on Firefox 3.6+ and Chrome/Safari 4.0+ currently.
    Opera.Next (Presto 2.4+) and IE 9 looks to get support for these features as well, but for older IE versions we'll need conditional comments to load CSS and graphic needed to make it look the same.


    Other ideas:
    1. Add context menu to subitems on far right side when you hover over the line (like dropbox webinterface)
